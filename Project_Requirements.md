# Project Requirements

## Deliverables

All must be submitted by the deadline:

- **PDF Report:** Single PDF document.
  - Font: Times New Roman, 12-point
  - Margins: 1 inch on all sides
  - Max 10 pages for main body (excluding cover page and appendix)
  - Appendix has no page limit — use for research sources, secondary figures, extended discussions
- **Oral Presentation:** ~8 minutes per team + 7-minute Q&A. Submit slides by the due date. Exact dates on syllabus.
- **Jupyter Notebook (.ipynb):** Well organized and commented. Must run with no errors.

> If you need to update your submission, re-upload all files — previous submissions are discarded. Only the final submission is graded.

---

## Grading Rubric / Roadmap

### Questions
- Formulate **two questions of interest** based on your dataset (predictive or prescriptive).

### Methods
- Use **at least two methods** from course tools (supervised or unsupervised):
  - Classification (e.g., KNN, Logistic Regression)
  - Regression (e.g., Decision Trees, Linear Regression)
  - Dimensionality Reduction (e.g., PCA)

### Data & Pipeline
- **Data:**
  - Define inputs and outputs
  - Identify data types (categorical vs. numerical)
  - Document all transformations and cleaning (normalization, missing values, one-hot encoding, thresholding, outliers, etc.)
  - Perform cross-validation or train/test split
- **Pipeline:**
  - Show the flow of information from input to output as a diagram (as shown in class)

### Evaluation
- Use the **right metric for the right task** (e.g., RMSE for regression, Confusion Matrix for classification)
- Accuracy alone is insufficient — address class imbalance and classification threshold selection
- Study model fitness: investigate overfitting, underfitting, or good fit
- **Tune at least two hyperparameters** to find the best performing model
- **Compare at least two approaches per question** (e.g., Decision Tree vs. KNN)

### Interpretation
- Don't just report numbers — explain findings and insights in plain English
- Write for a non-technical audience (e.g., upper management)
- Use visualizations where appropriate to understand data and results
